<task type="auto">
  <name>Task 1: Add update language endpoint (LANG-03, LANG-04)</name>
  <files>main_fastapi.py</files>
  <action>
    Add POST endpoint at `/api/v1/admin/languages/edit/{language_id}` that:
    1. Requires admin authentication via `Depends(get_current_admin_user)`
    2. Validates language_id exists (404 if not found)
    3. Validates required fields: language_code, language_name, is_active
    4. Updates all fields (code, name, is_active status)
    5. Catches IntegrityError for duplicate code or name (returns 409 Conflict)
    6. Returns updated language object

    Request body:
    ```json
    {
      "language_code": "en",
      "language_name": "English",
      "is_active": false
    }
    ```

    Response:
    ```json
    {
      "success": true,
      "message": "Language updated successfully",
      "language": {
        "id": 1,
        "language_code": "en",
        "language_name": "English",
        "is_active": false,
        "created_by": "admin",
        "created_at": "2026-02-27T..."
      }
    }
    ```

    Note: LANG-04 (toggle active status) is handled via this edit endpoint by setting is_active=false.

    Error handling:
    - 404 if language not found
    - 400 if required fields missing
    - 409 if language_code or language_name already exists (different language)
    - 500 on database errors

    Follow Flask pattern from modules/admin_languages.py lines 56-83.
  </action>
  <verify>
    <automated>curl -X POST http://localhost:8001/api/v1/admin/languages/edit/1 -H "Authorization: Bearer TOKEN" -H "Content-Type: application/json" -d '{"language_code":"en","language_name":"English","is_active":false}' | jq '.success'</automated>
  </verify>
  <done>Endpoint updates language with uniqueness validation, returns updated language</done>
